A local second-screen experiment for the Tesla browser.

ExtraDisplay keeps the signaling and controls on your Mac, uses the browser-native WebRTC path for realtime video, and stays focused on local network use.

01

Native Mac controller

Launch the SwiftUI app, start the local server, and check Screen Recording and Accessibility permissions in one place.

02

Tesla browser viewer

Open the URL shown by the app, usually http://<mac-ip>:8080, and view the selected Mac display.

03

Touch forwarding

Optional Accessibility access converts taps on the Tesla page into left-click events on macOS.

Setup path

The current prototype is designed for a Mac, an iPhone Personal Hotspot, macOS Internet Sharing, and the Tesla browser on the Wi-Fi network created by the Mac.

Create the local network

Connect iPhone to Mac over USB, enable Personal Hotspot, then share from iPhone USB to Wi-Fi in macOS Internet Sharing.

Launch ExtraDisplay

Open the macOS app, grant Accessibility permission if you want touch forwarding, and press Start.

Share the display

Use the sender page on the Mac to choose the display or experimental virtual display you want to stream.

Open it in Tesla

In the Tesla browser, visit the URL shown by ExtraDisplay and use the car screen as the viewer.

Prototype status: ExtraDisplay is intended for local use on your own Mac. Automatic car.display resolution and the native in-app sender pipeline are still in progress. The software virtual display path is experimental.
